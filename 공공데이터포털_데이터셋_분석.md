# 공공데이터포털 데이터셋 분석: 검진AI 리셋코치

## 최종 방향

공모전 출품 방향은 **건강검진 기반 AI 위험예측 및 퍼스널 개선 플랜 앱**으로 확정한다.

서비스명 가칭은 **검진AI 리셋코치**다.

> 국민건강보험공단 건강검진 공공데이터로 학습 가능한 AI가 고혈압·당뇨·이상지질혈증 위험도를 예측하고, 사용자의 생활패턴에 맞춰 작고 지속 가능한 개선 행동을 제안하는 개인 맞춤 예방관리 앱.

핵심 가치는 “진단”이 아니라 **위험예측과 예방 행동 유도**다. 사용자가 생활패턴을 조금만 바꿔도 위험 요인을 낮출 수 있다는 점을 체감하도록, 하루 1~2개의 작은 행동만 제안한다.

## AI 활용 방식

AI는 단순 요약 기능이 아니라 다음 역할을 담당한다.

1. 검진 수치 기반 질환별 위험도 예측
2. 위험도를 높인 주요 요인 설명
3. 사용자의 생활패턴을 반영한 퍼스널 개선 플랜 생성
4. 실천 난이도와 기대효과를 쉬운 말로 설명
5. 1주 단위 체크리스트 생성

예측 대상은 다음 3개로 제한한다.

- 고혈압 위험군
- 당뇨 위험군
- 이상지질혈증 위험군

## 라벨 생성 기준

MVP 단계에서는 공개 건강검진 데이터에 확정 진단 컬럼이 없다는 전제로, 건강검진 기준값을 활용해 위험군 라벨을 생성한다.

| 예측 대상 | 학습 라벨 기준 |
|---|---|
| 고혈압 위험군 | 수축기혈압 140 이상 또는 이완기혈압 90 이상 |
| 당뇨 위험군 | 공복혈당 126mg/dL 이상 |
| 이상지질혈증 위험군 | 총콜레스테롤, LDL, HDL, 중성지방 기준으로 위험 여부 생성 |

서비스 화면에서는 당뇨를 다음처럼 3단계로 보여준다.

| 단계 | 공복혈당 기준 |
|---|---|
| 정상 | 100mg/dL 미만 |
| 주의 | 100~125mg/dL |
| 위험 | 126mg/dL 이상 |

## 퍼스널 개선 플랜 원칙

추천은 사용자가 실생활에서 바로 할 수 있는 작은 행동으로 제한한다.

- 하루 1~2개 행동만 제안
- 위험 요인 1~2개만 우선 개선 대상으로 선택
- 운동은 식후 가벼운 산책, 계단 1층, 10분 걷기처럼 낮은 강도부터 시작
- 식단은 추가·교체·줄이기 중심으로 구성
- 식품 추천은 일반 예시로 표시하고, 알레르기·질환·약물 복용이 있으면 의료진 상담을 안내

금지 추천은 다음과 같다.

- 밥 먹고 바로 격한 러닝
- 하루 한 끼 또는 단백질 쉐이크만 섭취
- 특정 식품만 먹기
- 단기간 급격한 감량 목표
- 약 복용 중단 또는 의료 판단 대체

## 예시 시나리오

사용자 상황:

- 공복혈당 높음
- BMI 높음
- 운동 거의 없음
- 점심 후 달달한 커피를 자주 마심

앱 출력:

- “당뇨 위험군 가능성이 높게 예측되었습니다.”
- “위험도를 높인 주요 요인은 공복혈당, BMI, 운동 부족입니다.”
- 오늘의 작은 개선:
  - 점심 후 달달한 커피를 무가당 음료로 바꾸기
  - 저녁 식사 후 10분 천천히 걷기
  - 식사 시작 전에 방울토마토 5알 또는 채소 조금 먼저 먹기
- 1주 목표:
  - 단 음료 7잔 중 3잔 줄이기
  - 식후 걷기 3회 실천
  - 아침 또는 점심에 계란·두부 등 단백질 1가지 추가

## 비교 후보

| 후보 | 판단 |
|---|---|
| AI 사전문진 + 의료기관 매칭 | 서비스성은 높지만 의료기관 추천·증상 분류 중심이라 의료 판단 리스크 관리가 필요하다. |
| 약봉투 QR + AI 복약지도 | AI 활용성과 사회적 가치는 높지만 약국/조제정보 연계가 필요해 MVP 범위가 커질 수 있다. |
| 건강검진 기반 위험예측 및 개선 플랜 | 공공데이터 학습 구조가 명확하고, 예방관리 앱으로 제품화하기 쉽다. |

## 최종 선택 이유

건강검진 기반 위험예측 및 개선 플랜은 공공데이터 활용이 명확하고, AI가 위험예측과 개인화 추천의 핵심 역할을 수행한다. 또한 의료 진단이 아니라 예방적 건강관리 보조 서비스로 설계할 수 있어 공모전 시제품으로 현실성이 높다.

## 제출 및 연동 전략

공모전 제출은 **웹앱 제출 + 수동 입력**으로 진행한다. 앱스토어/플레이스토어 승인 절차가 필요한 네이티브 앱은 일정상 제외한다.

MVP 입력 방식은 다음과 같다.

- 건강검진 결과지의 주요 수치 직접 입력
- 최근 7일 평균 걸음수 직접 입력
- 생활패턴 직접 입력

향후 확장 로드맵은 다음과 같다.

| 단계 | 기능 | 설명 |
|---|---|---|
| MVP | 수동 입력 | 검진 수치, 걸음수, 생활패턴을 사용자가 직접 입력 |
| Next | OCR 업로드 | 건강검진 결과지 PDF/이미지에서 공복혈당, 혈압, 콜레스테롤 등 주요 수치 자동 추출 |
| Future | HealthKit/Health Connect | 사용자 동의 기반 걸음수, 활동량, 수면 등 자동 연동 |
| Future | 의료 마이데이터 | 건강정보고속도로 또는 공공 마이데이터를 통한 건강검진 내역 연동 |

현재 제출 버전에서는 HealthKit/Health Connect를 직접 붙이지 않고, **최근 7일 평균 걸음수 입력 필드**로 활동량을 반영한다.
