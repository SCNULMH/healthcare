# 분석모델 확인 기록

확인일: 2026-05-21

## 1. 현재 앱에서 실제 사용하는 모델

현재 배포 웹앱의 `/risk/predict`는 `app/services/diagnosis.py`의 설명 가능한 규칙 기반 위험예측 엔진을 사용합니다.

예측 대상:

- 당뇨 위험군
- 고혈압 위험군
- 이상지질혈증 위험군

입력값:

- 건강검진 수치: 나이, 성별, 키, 몸무게, 허리둘레, 혈압, 공복혈당, 지질 수치
- 생활패턴: 단 음료, 야식, 운동 빈도, 외식, 수면, 걸음수, 흡연, 음주, 식사 준비 가능 여부

출력:

- 질환별 위험도
- 위험도를 높인 주요 요인
- 하루 1~2개의 개선 행동
- 1주 목표
- 의료 진단이 아니라는 안전 문구

## 2. 규칙 기반 위험예측 기준

당뇨:

- 공복혈당 126 mg/dL 이상이면 큰 가중치
- 공복혈당 100~125 mg/dL이면 주의 가중치
- BMI, 허리둘레, 단 음료, 운동 빈도, 평균 걸음수 반영

고혈압:

- 수축기혈압 140 이상 또는 이완기혈압 90 이상이면 큰 가중치
- 수축기혈압 130~139 또는 이완기혈압 80~89이면 주의 가중치
- BMI, 외식, 음주, 운동 빈도, 평균 걸음수 반영

이상지질혈증:

- 총콜레스테롤 240 이상, LDL 160 이상, 중성지방 200 이상, HDL 40 미만이면 위험 가중치
- 총콜레스테롤 200 이상, LDL 130 이상, 중성지방 150 이상, HDL 50 미만이면 주의 가중치
- BMI, 외식, 야식, 운동 빈도, 평균 걸음수 반영

## 3. 학습 파이프라인 확인

학습 스크립트:

```powershell
python scripts/train_risk_models.py --csv .\data\health_checkup.csv --out .\models\risk_models.joblib
```

구현 내용:

- 국민건강보험공단 건강검진정보 CSV 로드
- 한글 컬럼명을 내부 표준 컬럼명으로 정규화
- BMI 계산
- 당뇨·고혈압·이상지질혈증 라벨 생성
- RandomForest 또는 LogisticRegression 학습
- ROC-AUC, classification report 저장
- 학습 불가능한 타깃은 실패 대신 `skipped` 리포트로 기록

## 4. 실제 데이터 확인 결과

공공데이터 API 연결 상태:

- `PUBLIC_DATA_SERVICE_KEY` 설정 확인
- `국민건강보험공단_건강검진정보` 샘플 조회 가능
- API 응답 기준 전체 건수: 1,000,000건

추가 확인:

- 건강검진정보 3,000건을 내려받아 학습 테스트 완료
- 당뇨 위험군 모델: 학습 가능
- 고혈압 위험군 모델: 학습 가능
- 이상지질혈증 위험군 모델: 현재 표본에서 지질 수치가 비어 있어 학습 제외

주의:

당뇨와 고혈압의 성능 지표가 높게 나오는 것은 공복혈당과 혈압 기준으로 만든 라벨을 같은 검진 수치로 분류하기 때문입니다. 따라서 제출서류에는 이를 "향후 질병 발생 예측 성능"으로 과장하지 않고, "건강검진 기준 위험군 분류 및 설명 가능한 예방관리 MVP"로 설명하는 것이 안전합니다.

## 5. 제출용 표현

현재 MVP는 규칙 기반 위험예측 엔진으로 안정적으로 동작합니다. 별도 학습 스크립트를 통해 건강검진정보 CSV 기반 RandomForest/LogisticRegression 모델 학습 파이프라인도 구현했으며, 공복혈당과 혈압 기반 위험군 분류는 실제 공공데이터 표본으로 학습 가능함을 확인했습니다. 다만 이상지질혈증은 현재 API 표본의 지질 수치 결측으로 인해 MVP에서는 규칙 기반 산출을 유지합니다.
