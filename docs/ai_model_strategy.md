# AI·모델 활용 설명

## 1. 현재 MVP의 AI 활용 성격

현재 MVP는 사용자가 입력한 건강검진 수치와 생활패턴을 바탕으로 고혈압·당뇨·이상지질혈증 위험도를 예측하고, 위험 요인에 맞는 개인별 개선 행동을 생성합니다.

MVP 위험예측은 규칙 기반 점수화 방식으로 구현되어 있습니다. 공복혈당, 혈압, 지질 수치, BMI, 허리둘레, 운동 빈도, 단 음료 섭취, 걸음수 등 질환별 주요 위험 요인에 가중치를 부여해 위험도를 계산합니다.

심사용으로는 AI 활용을 3단계로 설명합니다.

1. 위험도 예측 AI: 건강검진 수치와 생활패턴을 바탕으로 질환별 위험도를 산출
2. 설명 AI: 위험도를 높인 요인을 쉬운 말로 설명
3. 퍼스널 플랜 생성 AI: 위험요인을 하루 1~2개의 실천 가능한 행동으로 변환

AI의 핵심 역할은 단순히 위험도를 예측하는 것이 아니라, 공공데이터 기반 위험요인을 사용자가 실행 가능한 예방관리 행동으로 바꾸는 것입니다.

부가적으로 OCR 입력 보조 기능은 `OCR_PROVIDER=openai` 설정 시 OpenAI Vision을 사용해 건강검진 결과지 이미지/PDF에서 주요 수치를 JSON으로 추출합니다. 이 기능은 위험예측 자체가 아니라 사용자의 수동 입력 부담을 줄이는 보조 AI로 설명합니다.

이 방식을 사용한 이유는 다음과 같습니다.

- 심사 시 결과 해석이 명확함
- 데이터가 부족한 초기 시연에서도 안정적으로 동작함
- 어떤 요인이 위험도를 높였는지 설명 가능함
- 이후 머신러닝 모델로 교체해도 API와 UI 구조를 유지할 수 있음

## 2. 예측 대상

### 당뇨 위험군

주요 입력값:

- 공복혈당
- BMI
- 허리둘레
- 단 음료 섭취 빈도
- 운동 빈도
- 평균 걸음수

라벨 생성 기준:

- 공복혈당 126 mg/dL 이상이면 당뇨 위험 라벨 1
- 공복혈당 100~125 mg/dL 구간은 주의 위험 신호로 반영

### 고혈압 위험군

주요 입력값:

- 수축기혈압
- 이완기혈압
- BMI
- 외식 빈도
- 음주 수준
- 운동 빈도
- 평균 걸음수

라벨 생성 기준:

- 수축기혈압 140 mmHg 이상 또는 이완기혈압 90 mmHg 이상이면 고혈압 위험 라벨 1
- 수축기혈압 130~139 또는 이완기혈압 80~89 구간은 주의 위험 신호로 반영

### 이상지질혈증 위험군

주요 입력값:

- 총콜레스테롤
- LDL
- HDL
- 중성지방
- BMI
- 외식 빈도
- 야식 빈도
- 운동 빈도

라벨 생성 기준:

- 총콜레스테롤 240 mg/dL 이상
- LDL 160 mg/dL 이상
- 중성지방 200 mg/dL 이상
- HDL 40 mg/dL 미만

위 조건 중 하나라도 해당하면 이상지질혈증 위험 라벨 1로 생성합니다.

## 3. 학습 파이프라인

학습 스크립트는 `scripts/train_risk_models.py`에 구현되어 있습니다.

흐름:

1. 국민건강보험공단 건강검진정보 CSV 로드
2. 한글 컬럼명을 내부 표준 컬럼명으로 정규화
3. BMI 파생 변수 생성
4. 질환별 라벨 생성
5. 수치형 변수는 StandardScaler 적용
6. 범주형 변수는 OneHotEncoder 적용
7. RandomForestClassifier 또는 LogisticRegression 학습
8. 질환별 모델과 성능 리포트 저장

현재 공공데이터 API 표본 확인 결과, 당뇨와 고혈압 라벨은 공복혈당·혈압 수치로 생성해 학습할 수 있습니다. 다만 일부 건강검진정보 표본에서는 총콜레스테롤, HDL, LDL, 중성지방 값이 비어 있어 이상지질혈증 학습 라벨을 만들 수 없습니다. 이 경우 학습 스크립트는 실패하지 않고 해당 타깃을 `skipped`로 기록하며, 앱 MVP는 이상지질혈증을 규칙 기반 엔진으로 계속 산출합니다.

중요한 해석상 주의점:

- 현재 라벨은 검진 수치 기준으로 생성한 위험군 라벨입니다.
- 따라서 모델 성능은 "향후 질병 발생 예측"이 아니라 "검진 기준 위험군 분류 재현" 성격입니다.
- 실제 장기 예후 예측 모델로 확장하려면 추적 관찰 데이터, 진단 이력, 약물 복용 이력, 의료 마이데이터 연계가 필요합니다.

실행 예시:

```powershell
python scripts/fetch_public_data.py --dataset checkup --pages 1 --per-page 1000 --out .\data\health_checkup.csv
python scripts/train_risk_models.py --csv .\data\health_checkup.csv --out .\models\risk_models.joblib
```

## 4. 모델 후보

### 1차 모델: Logistic Regression

장점:

- 설명 가능성이 높음
- 변수별 영향 방향을 설명하기 쉬움
- 심사 자료에서 AI 판단 근거를 제시하기 좋음

단점:

- 비선형 패턴 반영이 제한적

### 2차 모델: Random Forest

장점:

- 비선형 관계 반영 가능
- 변수 중요도 산출 가능
- 건강검진 수치 조합에 따른 위험 패턴을 더 유연하게 반영

단점:

- Logistic Regression보다 설명이 복잡함

## 5. MVP와 학습 모델의 연결 구조

현재 앱 구조는 라우터와 UI가 `RiskRequest`와 `RiskResult` 형태만 바라보도록 구성되어 있습니다. 따라서 내부 위험 엔진을 규칙 기반에서 학습 모델 기반으로 바꾸더라도 프론트엔드와 API 계약을 유지할 수 있습니다.

교체 방식:

- 현재: `app/services/diagnosis.py`의 규칙 기반 점수화
- 향후: `models/risk_models.joblib` 로드 후 질환별 예측 확률 계산
- 유지: `/risk/predict` API 응답 구조

## 6. 퍼스널 플랜 생성 방식

퍼스널 플랜은 위험도가 높은 질환 1~2개를 우선 대상으로 선택한 뒤, 사용자의 생활패턴을 반영해 하루 1~2개의 작은 행동만 추천합니다.

예시:

- 공복혈당 높음 + 단 음료 많음: 단 음료 1잔 무가당으로 교체
- 공복혈당 높음 + 운동 부족: 식후 10분 천천히 걷기
- BMI 높음 + 아침식사 불규칙: 계란·두부 등 단백질 1가지 추가
- 혈압 높음 + 외식 많음: 국물 절반 남기기
- 지질 수치 높음 + 야식 많음: 야식 1회 줄이기

추천 제한:

- 하루 1~2개 행동만 제안
- 사용자의 운동 경험이 낮으면 고강도 운동 제외
- 특정 식품만 먹는 방식 제외
- 약물 복용 중단 또는 치료 판단 제외

## 7. 제출 시 표현 문장

공모전 제출서류에는 다음과 같이 표현하는 것이 적절합니다.

> 본 서비스는 국민건강보험공단 건강검진정보의 항목 구조와 질환별 기준값을 바탕으로 고혈압·당뇨·이상지질혈증 위험도를 예측하는 MVP를 구현하였다. 현재 시연 버전은 설명 가능한 규칙 기반 위험 엔진을 사용하며, 별도 학습 스크립트를 통해 건강검진정보 CSV 기반 RandomForest/LogisticRegression 모델로 확장 가능한 구조를 갖추었다. 예측 결과는 단순 위험도 표시에서 끝나지 않고, 사용자의 생활패턴을 반영해 하루 1~2개의 실천 가능한 개선 행동으로 변환된다.

> AI 활용은 위험도 예측, 위험 요인 설명, 개인 맞춤 개선 플랜 생성의 3단계로 구성된다. 현재 MVP는 설명 가능한 규칙 기반 AI 엔진으로 안정적으로 동작하며, 향후 학습 모델을 `/risk/predict` 내부 엔진으로 교체해도 UI와 API 구조를 유지할 수 있다.

## 8. 남은 모델 작업

- 건강검진정보 CSV를 충분한 규모로 다운로드
- 결측치 처리 기준 확정
- 질환별 라벨 분포 확인
- Logistic Regression과 Random Forest 성능 비교
- ROC-AUC, recall, precision 등 지표 산출
- 변수 중요도 또는 계수 기반 설명 자료 작성
- 학습된 모델을 `/risk/predict` 내부 엔진으로 선택 적용

## 9. 런타임 모델 분기

앱은 `RISK_MODEL_MODE` 환경변수로 예측 엔진을 선택합니다.

- `auto`: 학습 모델 파일이 있으면 사용하고, 없으면 규칙 기반 엔진으로 자동 전환
- `rule`: 항상 규칙 기반 엔진 사용
- `model`: 학습 모델 파일 로드 실패 시 503 오류 반환

기본 배포는 `auto`가 안전합니다. Render 무료 환경처럼 모델 파일을 배포 파일시스템에 올리기 어려운 경우에는 자동으로 규칙 기반 엔진을 사용합니다. Colab 등 외부 환경에서 생성한 `risk_models.joblib`를 서버에 배치하고 `RISK_MODEL_PATH`를 맞추면 학습 모델을 우선 적용할 수 있습니다.
