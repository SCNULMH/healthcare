## 모바일 앱형 UI 재교체

### 수행 내용
- 사용자가 제공한 UXPilot 템플릿 흐름을 기준으로 UI 교체 작업을 다시 수행.
- 기존 랜딩형/대시보드형 단일 페이지 구조를 폐기.
- `static/index.html`을 모바일 앱 프레임 구조로 재작성.
- 화면을 다음 단위로 분리.
  - 홈
  - 기본정보
  - 검진 데이터
  - 활동량
  - 생활패턴
  - OCR 업로드
  - 분석 결과
- 화면 하단에 고정 네비게이션 바 추가.
  - 홈
  - 입력
  - OCR
  - 분석
- 하단 네비게이션과 화면 내부 버튼으로 각 화면을 이동할 수 있도록 구현.
- `static/styles.css`를 모바일 앱 컨테이너, 카드형 입력, 단계 진행바, 하단 탭바 중심으로 전면 교체.
- `static/app.js`를 화면 전환 방식에 맞게 재작성.
- 기존 기능은 유지.
  - 데모 값 불러오기
  - 공공데이터 샘플 불러오기
  - OCR 데모
  - `/risk/predict` 위험예측
  - 결과 화면 렌더링

## Render 자동 배포 설정 보강

### 수행 내용
- Render 자동 배포가 여전히 동작하지 않는다는 사용자 피드백 확인.
- Render 문서 기준 기존 서비스는 Dashboard의 Auto-Deploy 설정이 유지될 수 있음을 확인.
- `render.yaml`에 `autoDeployTrigger: commit`만 두지 않고, 명시적으로 `autoDeploy: true`도 추가.
- 이후 Blueprint 재동기화 또는 신규 반영 시 자동 배포가 켜지도록 설정을 보강.

### 사용자 확인 필요
- Render Dashboard의 서비스 Settings에서 Auto-Deploy가 `Yes` 또는 enabled 상태인지 확인.
- 기존 서비스가 Blueprint 변경을 즉시 반영하지 않으면 Dashboard에서 직접 Auto-Deploy를 켜야 한다.
- 이번 1회는 Manual Deploy > Deploy latest commit으로 최신 커밋을 배포하는 것이 가장 빠르다.
